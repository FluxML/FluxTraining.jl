Visualize the callback dependency graph

You can use GraphPlot.jl to visualize the dependencies between callbacks:

learner = Learner(
    nothing, (nothing, nothing), nothing, nothing,  # dummy arguments
    ToGPU(),
);

using GraphPlot
gplot(learner.callbacks.graph, nodelabel = learner.callbacks.cbs, layout = stressmajorize_layout)

(the target of an arrow depends on the origin)

As an example for a detected dependency, we can see that Recorder runs after Loss. Recorder records the values of all metrics, so Loss which is a subtype of FluxTraining.AbstractMetric needs to run first.
